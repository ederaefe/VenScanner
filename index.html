Vulnerability Scanning and Threat Detection in Plain English

A private, secure alternative to cloud-upload virus scanners. Run static checks and AI code assessments directly on your device.

Local File Inspection

We read files directly in the browser. The scanner evaluates content locally on your computer. No file data is sent to external servers, protecting your privacy.

File Format Verification

We inspect the internal headers of files. The scanner checks if a program or script is hiding behind a safe extension like an image or text document.

AI Code Security Auditing

We scan script files to find bugs and flaws. The scanner analyzes code to find security errors that could allow injections or leaks.

1. Cryptographic SHA-256 Hashing

Generates unique file signatures locally in the browser utilizing the Web Crypto subtle digest standard. Large files yield thread execution back to the browser layout queue to prevent frame-blocking.

2. Binary Slicing Analysis

Checks headers (magic bytes) and standard signatures (like EICAR strings) by slicing only the first 2KB of the file. This keeps memory allocation near zero, preventing browser crash thresholds.

3. Multi-Tiered AI Code Auditing

Coordinates prompts targeting Chrome's window.ai.languageModel, falling back to secure Vercel Edge Serverless functions (Gemini 2.5 Flash API) and local AST-pattern regex matching.

4. Secure Header Edge Sandbox

Vercel Edge functions protect API execution by returning strict response headers (X-Content-Type-Options: nosniff, X-Frame-Options: DENY, and sandbox Content-Security-Policy rules).

Frequently Asked Questions

How does a browser-based scanner detect malware?
VenScan scans files in three layers. First, it calculates the file's SHA-256 hash locally and queries a secure cloud lookup api to check VirusTotal database reputation. Second, it runs header analyses on binary signatures (like EICAR test scripts) and magic bytes. Third, it audits text files for scripting anomalies.
Is it safe to run code files through the AI scanner?
Yes. Because the AI scanner prefers Chrome's built-in window.ai API, all neural-network evaluations happen on your device's graphics card or CPU—completely offline. If you trigger the cloud AI fallback, only the code snippet is sent to a secure, ephemeral Gemini endpoint, and is never used for training.
What file formats are supported?
VenScan accepts all file types up to 100MB. It runs specialized structural and AI parsing on scripting languages (.js, .py, .c, .cpp, .java, .php, etc.) and checks headers on PDF, zip archives, and binaries.
